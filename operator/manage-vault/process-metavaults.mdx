# Process Meta Vaults

:::custom-warning[V4 Only]
Process Meta Vaults is only available for Operator Service V4. [Upgrade →](/operator/operator-service-v3/v4-upgrade-guide)
:::

:::custom-info[Prerequisites]
- **Operator installed**: Follow the [Installation →](/operator/launch-operator-service-v4/installation) guide to set up the Operator Service
- **Wallet ready**: The operator needs a wallet to pay gas for transactions. See [Wallet Setup →](#wallet-setup) below.
:::

### Wallet Setup

#### Option A — Create a new wallet

Initialize a mnemonic with:
```bash
./operator init
````
Then create a wallet with:
```bash
./operator create-wallet
```

#### Option B — Use an existing wallet

<img src={require('./img/export-private-key-720.gif').default} alt="export-private-key" style={{maxWidth: '500px', width: '100%', borderRadius: '12px'}} />

Export a private key from an existing wallet, then set it as an environment variable:

```bash
export WALLET_PRIVATE_KEY="0xyour_private_key_here"
```

## `process-meta-vaults` Command

The `process-meta-vaults` command triggers transactions to update states and process deposits and exits for one or more Meta Vaults.

### Command Usage

```bash
./operator process-meta-vaults [OPTIONS]
```

:::custom-warning[Required Parameters]
The following parameters are required:
- `--vaults` - Comma separated addresses of meta vaults to process
- `--execution-endpoints` - Comma separated API endpoints for execution nodes. You can find public RPC endpoints on [Chainlist ↗](https://chainlist.org/chain/1)
- If using [Option A →](#wallet-setup): `--wallet-file` and `--wallet-password-file` flags
- If using [Option B →](#wallet-setup): the `WALLET_PRIVATE_KEY` environment variable is picked up automatically
:::

### Optional Flags

| Flag | Description |
|------|-------------|
| <nobr>`--network`</nobr> | The network: `mainnet`, `hoodi`, or `gnosis` |
| <nobr>`--graph-endpoint`</nobr> | API endpoint for graph node |
| <nobr>`--min-deposit-amount-gwei`</nobr> | Minimum amount in gwei that must accumulate in the vault to trigger deposits to the sub-vaults. Default: 10,000,000,000 Gwei (10 ETH/GNO) |
| <nobr>`--max-fee-per-gas-gwei`</nobr> | Maximum fee per gas for transactions. Default: 10 Gwei for Ethereum, 2 Gwei for Gnosis |
| <nobr>`-v, --verbose`</nobr> | Enable debug mode. Default: false |

## Example

```bash
./operator process-meta-vaults \
 --vaults=0x15639E82d2072Fa510E5d2b5F0db361c823bCad3 \
 --wallet-file=/Users/user/wallet.json \
 --wallet-password-file=/home/user/wallet-pass.txt \
 --execution-endpoints=https://execution-node
```
