# Create a Regular Vault

A Regular Vault runs validators and stakes assets.

### Step 1: Select Vault Type

<ThemedImage
  alt="step1"
  sources={{
    light: require('./img/step1-light.png').default,
    dark: require('./img/step1-dark.png').default,
  }}
  style={{maxWidth: '600px', width: '100%'}}
/>

Select **Regular Vault**. You can optionally enable the following:

- **<img src="/img/lock.png" alt="lock" style={{width: '24px', height: '24px', display: 'inline', verticalAlign: 'middle'}} /> Private** — restrict deposits to approved wallets only
- **<img src="/img/stop.png" alt="stop" style={{width: '22px', height: '22px', display: 'inline', verticalAlign: 'middle'}} /> Block List** — prevent specific addresses from depositing
- **<img src="/img/rewards.png" alt="rewards" style={{width: '24px', height: '24px', display: 'inline', verticalAlign: 'middle'}} /> ERC-20 Token** — stakers receive a token they can hold in their wallet and transfer to others

:::custom-warning[Required Acknowledgment]
You must check **"I understand that the selected vault type cannot be changed later"** before proceeding.
:::

### Step 2: Configure Vault Parameters

<ThemedImage
  alt="step2"
  sources={{
    light: require('./img/step2-light.png').default,
    dark: require('./img/step2-dark.png').default,
  }}
  style={{maxWidth: '600px', width: '100%'}}
/>

Configure your Vault parameters:

- **Vault Capacity** – Set maximum ETH capacity (default: unlimited ∞). [Learn more →](/docs/vaults/customization#capacity)
- **Vault Fee** – Set the fee percent your Vault will charge from users staking rewards

:::custom-info[Vault Fee]
The Vault Fee can be updated after creation. [Learn more about fee management →](/docs/fees/intro#vault-fees)
:::

If you enabled the **ERC-20 Token** in [Step 1 →](#step-1-select-vault-type), you will also need to set:
- **Token Name** – Your Vault ERC-20 token name
- **Token Symbol** – Your Vault ERC-20 token symbol

:::custom-warning[Permanent Settings]
All parameters above, except the fee, **cannot be changed later**. Choose carefully before proceeding.
:::

### Step 3: Block Rewards Destination

<ThemedImage
  alt="step3"
  sources={{
    light: require('./img/step3-light.png').default,
    dark: require('./img/step3-dark.png').default,
  }}
  style={{maxWidth: '600px', width: '100%'}}
/>

Choose how block rewards are distributed. This is a **permanent decision**.

Most operators choose [Smoothing Pool →](/docs/vaults/customization#smoothing-pool--pooled-rewards) for stable returns, while [Vault Escrow →](/docs/vaults/customization#own-escrow--own-rewards) gives you full control but with volatile rewards.

:::custom-warning[Required Acknowledgment]
You must check **"I understand that block rewards destination cannot be changed later"** before proceeding.
:::

### Step 4: Add Branding

<ThemedImage
  alt="step4"
  sources={{
    light: require('./img/step4-light.png').default,
    dark: require('./img/step4-dark.png').default,
  }}
  style={{maxWidth: '600px', width: '100%'}}
/>

Customize your Vault's appearance with name, description, and logo.

:::custom-notes[Branding Can Be Changed Later]
Unlike Vault parameters, you can update branding anytime via your Vault interface: press **Settings → Branding → Edit → Save**.
:::

### Step 5: Summary

<ThemedImage
  alt="step5"
  sources={{
    light: require('./img/step5-light.png').default,
    dark: require('./img/step5-dark.png').default,
  }}
  style={{maxWidth: '600px', width: '100%'}}
/>

Review all your settings before final submission.
This is your last chance to verify everything is correct, such as name, capacity, fees, type, and MEV strategy.

:::custom-warning[Risk Acknowledgment]
You must check **"I understand the risks of operating the Vault and confirm my configuration choices"** before proceeding.
:::

Click **Create Vault** to submit your configuration and create the Vault.

### Step 6: Transaction Confirmation

Confirm the transaction in your wallet to deploy your Vault.
Once the transaction confirms, your Vault is live and visible on the **Operate** page! 🎉

:::custom-info[Next Steps After Creation]

**Set up validators** – Essential for Vault operation. Use the [Setup Checklist →](./getting-your-vault-running) to ensure all requirements are met.

**Reach out** to the [StakeWise team](mailto:info@stakewise.io) for Vault verification (better visibility and trust with potential stakers)
:::
