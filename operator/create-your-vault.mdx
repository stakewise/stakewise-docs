# Create Your Vault

Now that you know what operating a Vault involves, let's create one.

Creating a Vault is done via the StakeWise UI and takes just a few minutes. If you're not sure about the exact parameters for your Vault, reach out to the [StakeWise team](mailto:info@stakewise.io) for personalized guidance. Once your Vault is created, you can set up the validator infrastructure and launch the Operator Service to begin your operator journey.

## Getting Started

1. Navigate to the [StakeWise App ↗](https://app.stakewise.io) and go to the **Operate** page
2. Click the **Create Vault** button to begin the setup process

:::custom-notes[Test on Hoodi First]
To try the full Vault setup on a testnet, open **Settings** <img src="/icons/stakewise/gear_icon.png" alt="gear button icon" style={{height: '2.5em', verticalAlign: 'middle', display: 'inline-block'}} /> in the StakeWise App and enable **Show Testnets**. Then switch to the Hoodi network to create and test your Vault before going to mainnet.

You'll need some Hoodi ETH to cover gas fees. Get testnet ETH from a faucet:
- [hoodi-faucet.pk910.de ↗](https://hoodi-faucet.pk910.de/)
- [Google Cloud Hoodi Faucet ↗](https://cloud.google.com/application/web3/faucet/ethereum/hoodi)
:::

## Step 1: Configure Vault Parameters

Configure your Vault with the following settings:

- **Vault Capacity** – Set [maximum ETH capacity](/docs/vaults/customization#capacity) (default: unlimited ∞).
- **Vault Fee** – Set the fee percentage your Vault will charge from [staking rewards](/docs/fees/intro#vault-fees).
- **Vault Token** – Enable a tradeable ERC-20 token (requires name & symbol)
- **Private Vault** – Restrict to approved wallets only
- **Block List** – Prevent specific addresses from depositing

:::custom-warning[Settings After Creation]
The **Vault fee** can be updated after creation via **Settings → Vault fee → Edit → Save**. If you've made your Vault private or added a block list, you can add/remove authorized addresses via **Vault access → Edit → Save**.
:::

## Step 2: Block Rewards Destination

Choose how block rewards are distributed. This is a **permanent decision**.

Most operators choose Smoothing Pool for stable returns, while Vault Escrow gives you full control but with volatile rewards. See [MEV Strategy →](/docs/vaults/customization#mev-strategy) for details.

## Step 3: Add Branding

Customize your Vault's appearance with name, description, and logo.

:::custom-notes[Branding Can Be Updated]
Update branding anytime via **Settings → Branding → Edit → Save**.
:::

## Step 4: Summary

Review all your settings before final submission. This is your last chance to verify everything is correct, including name, type, MEV strategy, fees, and capacity.

Click **Create Vault** to submit your configuration and create the Vault.

## Step 5: Transaction Confirmation

Confirm the transaction in your wallet to deploy your Vault.
Once the transaction confirms, your Vault is live and visible on the **Operate** page.
