# Prerequisites

Before installing and configuring the Operator Service, your node infrastructure must meet specific technical requirements to ensure reliable operation.

## Execution Client

Your execution node must be fully synced and running.
Any execution client that supports the [ETH Execution API specification 竊余(https://ethereum.github.io/execution-apis/api-documentation/) can be used:

<Tabs groupId="execution-clients">
  <TabItem value="nethermind" label="Nethermind" default>
    **[Nethermind 竊余(https://launchpad.ethereum.org/en/nethermind)** - Supports both Ethereum and Gnosis networks

    Standard installation and configuration. No special requirements.
  </TabItem>
  <TabItem value="besu" label="Besu">
    **[Besu 竊余(https://launchpad.ethereum.org/en/besu)** - Ethereum network support

    Standard installation and configuration. No special requirements.
  </TabItem>
  <TabItem value="erigon" label="Erigon">
    **[Erigon 竊余(https://launchpad.ethereum.org/en/erigon)** - Supports both Ethereum and Gnosis networks

    :::warning Special Configuration Required
    Event logs must be preserved for the Operator Service to function properly:

    ```bash
    # Ethereum Mainnet
    erigon --prune=receipts --prune.to=21471500

    # For other networks, adjust block number to Keeper contract creation block
    ```

    The required flags prevent event log pruning that the Operator reads from certain contracts.
    :::
  </TabItem>
  <TabItem value="geth" label="Geth">
    **[Geth 竊余(https://launchpad.ethereum.org/en/geth)** - Ethereum network support

    Standard installation and configuration. No special requirements.
  </TabItem>
  <TabItem value="reth" label="Reth">
    **[Reth 竊余(https://github.com/paradigmxyz/reth)** - Ethereum network support

    <Admonition type="warning" icon={<img src={useBaseUrl('/img/exclamation_mark.png')} alt="Warning" style={{ width: 20, height: 20 }}/>} title="Special Configuration Required">
      The problem with Reth could be event logs pruning. Operator reads event logs from certain contracts. In order to preserve those logs execution client needs to be tweaked.

      The required setting in the `reth.toml` is:

      ```toml
      [prune.segments.receipts_log_filter.0x6b5815467da09daa7dc83db21c9239d98bb487b5]
      before = 21471500
      ```

      Here `0x6b5815467da09daa7dc83db21c9239d98bb487b5` is [Keeper contract 竊余(https://etherscan.io/address/0x6B5815467da09DaA7DC83Db21c9239d98Bb487b5#code) address on Ethereum Mainnet. Note that the address must be lowercased.
    </Admonition>
  </TabItem>
</Tabs>
