## File Structure

- `operator/` — current Operator docs (v4.x)
- `operator-v3/` — legacy Operator docs (v3.x)

## Code review conventions

Every time you perform a code review, make sure that the files with the .md and .mdx extensions in the docs/sdk folder include a title and a description.


## Git Conventions

Use short commit messages only (no description or co-author).

Leave the pull request summary empty when opening a PR, as GitHub Copilot will automatically generate the summary.
