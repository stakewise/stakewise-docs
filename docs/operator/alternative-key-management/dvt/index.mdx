# DVT

Distributed Validator Technology (DVT) is a validator security mechanism that distributes key management and signing operations across multiple participants, avoiding a single point of failure and increasing validator resilience.
Validators have two public-private key pairs: active keys for consensus participation (block proposals, attestations) and withdrawal credentials that determine where staked funds can be sent upon exit.
While withdrawal credentials can remain offline, validator keys must stay online continuously, creating security risks if compromised.

DVT mitigates this by encrypting and splitting the original key into shares distributed across multiple nodes, allowing stakers to keep the master key in cold storage.

A threshold number of these shares (for example, 3 out of 4) can collectively produce the signatures needed for validator operations. The system relies on cryptographic components including distributed key generation, threshold signature schemes, and multiparty computation to ensure that no single node ever possesses the complete key.
